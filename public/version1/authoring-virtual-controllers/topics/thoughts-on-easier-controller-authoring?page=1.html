(Note: this maybe should be in the feedback forum, but since it is directly related to authoring virtual controls, I decided to put it here instead.)

After working on a couple of virtual controllers today I found myself doing the following work-flow: - Create the controller in Gameplay - Create some of the initial controls with glyphs and actions and roughly place them - Export the controller to a .xml file - Open that file in an editor and clean up a lot of the settings: make the transparencies consistent, align the controls, fix scaling, etc. - Delete the controller in Gameplay and re-import it - Make edits to the layout based on game play, etc. - Go back to the export stage - rinse, repeat until controller is working well. - If I wanted a second view, I often edited the .xml file to copy controls from one view to the other as a starting point.

A few thoughts after doing this a bunch: - It would be great if, on import, Gameplay fixed some of the obviously illegal stuff like duplicate IDs (I wasn't sure what effect this would have on the controller and it seemed to persist through an import/export, so I found myself making sure they were unique, but this seems like an easy thing to do during import). - It would also be great to make this editing "loop" easier: I found that the most annoying step was needing to delete the controller before re-importing it - something simple where re-importing an .xml file that one of the controllers was last loaded from or exported to triggered a prompt about "new or update" would be a big help. The most useful would be to have a button in the editor that exported the .xml file, launched the editor on the file, waited for the editor to close and then updated the controller layout from the .xml file. That would be just awesome. :) - Screenshot backgrounds for views were lost during the export/import process - the paths were in the .xml file so it was likely part of the delete and re-import that caused them to go away but it was a minor annoyance.

Obviously, you could make all of this unnecessary by improving the editor to have the ability to do things like type values into the opacity (and similar) fields, supporting copy/paste of controls, and having snap to grid or other alignment tools but making this workflow simpler/safer might be a way to make authoring more robust until the editor has all the needed features.

Thanks for reading! RainyDayHiker
