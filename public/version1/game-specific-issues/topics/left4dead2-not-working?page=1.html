Hi StalkingWolf,

To fix the error that you are getting in game, go into your controller settings in Gameplay, and change the Executable Path to point to just Steam itself (by default: C:\Program Files (x86)\Steam\Steam.exe)

Then, under Command Line Arguments, type in: -applaunch 550

The number is the Steam ID for Left 4 Dead 2.

This should allow you to connect in game without any problems. However, since you are now setting the Executable Path to just Steam itself, only the very top Virtual Controller you have on your list in Gameplay will show up if all your Controllers are visible. To get the game to play with the Controller you wish, first disable all the rest and then launch.

As for the drag gesture causing you to fire your gun continuously, try mapping that Input in the Global Gestures tab to be either Mouse Look or Mouse Hover. This should fix that issue.

Thanks, and let us know how it goes!

Just a clarification on this:

> since you are now setting the Executable Path to just Steam itself, only the very top Virtual Controller you have on your list in Gameplay will show up if all your Controllers are visible.

This is only applicable to other Virtual Controllers mapped to steam.exe itself.

When a game is launched, Gameplay loads the first enabled VC that it finds mapped to a particular executable. This way, if you have multiple VCs configured for the same game (same .exe), you can specify which one you wish to use by disabling all of the others for that game except for the one you want to load.

Thus, if you have multiple VCs that are configured using the -applaunch technique explained above by ArcaneAvis (executable set to steam.exe and command line args set to -applaunch APPID), then Gameplay loads the first enabled VC it finds mapped to steam.exe so you'll want to make sure to disable any other VCs you have configured this way before launching this one. If you have only one VC configured this way, you don't have to worry about it.

(NOTE: We've changed this a bit for the upcoming Gameplay 2 - launching a Virtual Controller will launch that VC even if you have another VC configured for the same .exe)
