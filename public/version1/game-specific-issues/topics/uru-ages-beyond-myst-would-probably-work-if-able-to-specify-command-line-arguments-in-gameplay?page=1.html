Hello,

Uru: Ages Beyond Myst will not work with the controller for the following reason: - the main Executable is UruExplorer.exe - When running UruExplorer.exe, it calls UruSetup.exe, and then kills itself immediately (i.e. UruExplorer is not running anymore) - When pressing the "Play" button in UruSetup.exe, UruExplorer.exe is launched again, this time with some specific command-line options (in my case -iinit To_Dni) - Of course the above method will completely kill any compatibility with Gameplay since the process dies once before being relaunched. GestureKeys works, as it can connect to a running process. - If I launch UruExplorer.exe -iinit To_Dni (through command line or batch file), then UruExplorer.exe will keep running all the time and no other process is launched, so my impression is that this probably would work fine with Gameplay (although I tried with the batch file containing the arguments, no luck)

However, Gameplay doesn't accept any command line arguments for the titles. It would be nice to have a way to add command-line arguments to the actual path, if possible.

I think this would greatly enhance compatibility with some games. After all, on a tablet, most titles being played will be somewhat legacy titles, which definitely can have some quirks.

Any thoughts are welcome!

Yannick

Thanks for all the feedback Yannick! As of v0.9.2.8, you can specify command line arguments to the target application in the Advanced section of the Setting menu for each Virtual Controller. Mouse scroll should also be functional now.

We've done a bit of testing with Uru and unfortunately it looks like although the game specifications lists DirectX 9.0c as the minimum requirement, it is actually loading the DirectX 8 libraries. This explains why there is no overlay, even when supplying the correct parameters.
