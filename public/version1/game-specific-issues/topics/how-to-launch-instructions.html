Having issues with launching games with default/custom Virtual Controllers? Before submitting a bug report, please make sure you try these five different launch scenarios. Included in this list are the launch scenarios for default VCs in GestureWorks.

1. Normal Launch: Launch the game from either the Steam/Origin client or the GestureWorks Gameplay tool with the Virtual Controller enabled.

Games with this Launch Sequence: Bastion, Castle Crashers, Castlevania Lords of Shadow, Dead Space, Diablo III, Guacamelee!, Legend of Dungeon, Lego Star Wars: The Complete Saga, Mark of the Ninja, Mass Effect 3, Mortal Kombat Komplete Edition, Need for Speed: Shift, Elder Scrolls V: Skyrim, Sonic and SEGA All-stars Racing, Tomb Raider

2. Gameplay Launch Only: Enable virtual controller in Gameplay tool. Launch the game from the Gameplay tool.

Games with this Launch Sequence: World of Tanks

3. Close Steam to Launch: Make sure that the Steam/Origin client is not running and, with Virtual Controller enabled, launch the game from Gameplay. Gameplay will launch Steam/Origin client.

Games with this Launch Sequence: Batman Arkham Asylum, Burnout Paradise, Dust An Elysian Tail, Fallout New Vegas, Just Cause 2, Kingdoms of Amalur, Portal 2, Red Faction Armageddon, Trackmania

4. Steam Launch Only: Login to Steam/Origin, and launch the game through the client.

Games with this Launch Sequence: Assassins Creed III, Terraria

5. Launch a Different Game: Launch a different game first, then launch the game you want to play from the Steam/Origin client and/or Gameplay with the Virtual Controller enabled.

Games with this Launch Sequence: Borderlands 2, Strike Suit Zero

Please lets us know if you have any questions!
