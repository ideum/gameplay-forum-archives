I submitted a Bug Report outlining the issue i experienced on my Venue 8 Pro, in my instance it discovered it trying to play Skyrim. I'll also put a copy of my bug report here incase anyone else happens across it.

On a Dell Venue 8 Pro Install GestureWorks Gameplay Install Steam Install Skyrim in Steam (I've seen reports that other FPS styles games that require a mouse do not work on this device as well) Mouse inputs do not work.

This seems to be an issue with the lack of a mouse existing in the devices hardware. No mouse is listed in the Devices or in Control Panel -> Mouse.

Problem can be overcome by adding a USB or Bluetooth mouse to the device and leaving it connected. This allows GestureWorks Gameplay app the control the mouse cursor using the virtual joystick of the gesture based mouse inputs.

The issue obviously is that by adding a mouse it defeats the purpose of having the GestureWorks Gameplay app. It would seem that a "Virtual Mouse Device" similar to what some VM Clients install, if included with the app, might overcome but I have not been able to test this personally.

I think inteks and fredgfgr might have found the solution for this. Please try these steps if you haven't and let me know if it works for you all:

  1. Swipe in from the right edge of the screen, and then tap Search.
  2. Tap or click Settings.
  3. In the search box, enter Change how your mouse works, and then tap or click Change how your mouse works.
  4. Under Control the mouse with the keyboard, select the Turn on Mouse Keys check box, and then click OK.
