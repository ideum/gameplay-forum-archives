Mithrot
5 months ago
 

I am going to try to make a constructive critique of the software, as I know the people behind GWG are working very hard for 2.0, but the state this current release was in is just too much to not offer my 2cents.

I legally purchased this software for my Dell Venue Pro 8, with the promise that GWG would let me control my games like a controller. It delivers, but the execution is so botched that at times, you'd be better off just using an actual controller. Lets start with the positives:

  • Trial was fair, lightweight and does its job to give you a taste of its features. This is the best way to deal with the temptation of piracy
  • Buying was easy and instant
  • Easy to use interface
  • Adding games was straightfoward
  • Community features are excellent and a great way to provide access to popular control configs for games
  • Controls, when available, are functional. You click on them and they work as if its native and being able to resize/color them is neat
  • In-game editor was a smart idea to allow flexibility during cut-scenes or times when things had to be moved.

And thats about it. Now on to the bad:

  • Resourcehog. You can feel this app was designed for a Surface Pro rather than the leaner Baytrail tablets, which means that 2GB of Ram and slower CPU will hurt as soon as you try to move buttons or have certain games (Portal) slowdown. This is a good way of cutting your potential audience whenever GMG and a game hits that RAM limit for whatever reason.
  • Inflexible UI. The GWG window will start breaking if you rotate the screen or run games in anything below 1280x720, or alt-tab. For multi-taskers, this gets annoying.
  • Buttons cannot be swiped inbetween to simulate multiple presses. This is a big problem for more complicated games, forcing you to tap them individually.
  • Serious lack of different control options. We have switches and mouse wheel, but only 1 kind of Dpad. Why?
  • Garbage mouse control. The joystick is awful, but trying to set it so panning the touchscreen moves the mouse has GWG think your swipes are micro-stutters. This mans games look like they're lagging when in reality its GWG being unable to smooth it out. Try a proper mouse emulator like TouchMousePointer and see the difference. Utter trash.

But the above are issues that can be eventually fixed. Except until we get into the UGLY:

  • Games are compatible as long as they are DX9 and GWG feels like it. That means on weaker systems, you cant enjoy older games without them being recent (indies) or hope they dont break for whatever reason. This seriously undermines any reason to use GWG reliably beyond tested games, its essentially up to you to see if it works.
  • Slow, slooooow updates. I paid $15 and the most recent update was adding in left/right keyboard command keys? Really? Not even a opt-in beta?

As a gamer, I rarely buy software for games. Why would you? Money mostly goes either to hardware or...games. I want to feel as if GWG was as good as an investment as the Steam games that update with free content and so far, there is a very poor sense of GWG improving. Only a news feed showed off v2 ...by showing off you using a separate touchscreen device to control. NO info on improvements. Guys, what are you doing?

So what we have here is a app that works as long as you're fine essentially being a beta-tester for a specific number of games and emulators - hell forget that, since only select people can test 2.0 whereas everyone else is stuck with 1.0 thats barely updated at all. Even apps like PPSSPP and TouchMousePointer does touchscreen heads-above-heels of what GWG offers and they were free as side projects.

I have no idea how much better 2.0 is, but I have very little faith to recommend anyone buying this in anticipation. See what 2.0 can do and hope the folks here prove me wrong and make the wait worth it. I genuinely hope you guys find success. Thanks for reading this far.

  • Resource hog:

Thanks for looking into this. I wasn't sure if you were aware of the new Baytrail tablets, but the direction Intel is going with native x86 PC in a small form-factor with battery-life very similar to Android honestly seems like the future of Tablet gaming. Having the team check it out opens up another audience next to Surface owners. But the performance NEEDS to be there too.

  • Inflexible UI:

Yes, I was referring to the application window. On a tablet you will be re-sizing, rotating and moving the screen constantly unlike Desktop users. GWG as of now wont let me resize the window by much, and rotating is a big no-no unless you like all the buttons to cram together. I can't even do what seemed like obvious choices, like getting rid of the community panel if I want to slim down the home screen. Or an options panel, which is relegated to the blue 'i' (information icon?). As a user, I want options and right now there isn't any to customize GWG.

But thats not the only reason. For atleast my Baytrail tablet I have a 12800x800 resolution and GWG HATES it. UI doesn't scale, so everything becomes small. This gets much worse when you try making/editing controls because the buttons gets pixellated, the logos on them look unreadable and its impossible to click on the color/key options without using TouchMousePointer to accurately click on them. GWG also hilariously breaks when you use the Windows Text Scaling to above 100%, elements balloon to ridiculous portions.

All in all, GWG's UI was not made for Tablets. It is a desktop application for Mouses and thats a problem for smaller tablets.

  • Buttons cannot be swiped between:

Ever used GBCDroid for your android phone? It not only allowed this, but even allowed you to set the 'deadzone' of how far your thumb can go before it stopped reading in-between presses.

Why would you need this? Lets look at a game like Super Meat Boy. I have 2 buttons, run & jump. In GWG, I can run. I can jump. But I cannot run AND jump to build momentum, since I cannot slide my thumb from the 'run' key to the 'jump' key to emulate this. This doesn't happen for the Dpad either. Hopefully now you understand, this can always be an option which was what GWG desperately lacks.

In fact, PPSSPP does it.Thinking about it now PPSSPP does alot of things way better. Ever used it?

Ask your head UI dev to do me a favor. Install PPSSPP 0.98 and either install a free demo (I recommend MGS Peace Walker) or make sure you own a PSP game to test out (not advocating piracy). Have him enable touch controls on his Surface, what do you think he'll see?

PPSSPP has a fullscreen, tablet-friendly launcher for all my games, just a click. The controls can be re-sized and moved with only my thumbs, no need for a mouse. The in-game controls feature big, responsive buttons that are easy to use and grow in size everytime you press them. The buttons' look can be customized by switching user-made skins. Its fast and responsive.This is all done by a free x86 emulator.

Have him and the rest of the team learn from PPSSPP, this is what GWG should be doing for all my DirectX games. Thats a convenience people will pay for.

  • Lack of different control options:

I want a free-form joystick that lets me not only use it for mouse/wasd, but for any button(s). Allow me to set the deadzones. Allow me to set if I want the joystick in 4/6/8 way directions. Allow me to set one half of the screen so whenever I move my thumb anywhere on that screen, the joystick will appear and move in that direction.

I want the buttons to grow a bit in size when I press them. I want custom skins. I want a hudget where its a 'slide' bar and when I slide my thumb left or right, buttons will be pressed. Like left/right, for example. I want a hudget that is literally a box I can resize, so if I set the box to take up one half of the screen, it'll (for example) reverse my car in Need for Speed. I want to set if buttons disappear/translucent after not being used for awhile. I want the GWG keyboard to appear on the screen smoothly when I press a button, like on Windows 8. I want to set a resize-able box where if I swipe/tap on that space, keys will be pressed. Kinda like the touchscreen gesture options.

This should give you an idea of what I saw in different android emulators/games. There are thousands of PC Games, GWG had to accommodate like a swiss army knife, even if its pointless.

  • Garbage mouse control:

I only tested this on Source games since finding a DirectX 9 FPS that'll run on my tablet and still work on GMG is too specific to try out. This also happened on Killing Floor.

Install Portal, set some basic WASD and M1/M2 buttons then have the rest of the touchscreen set to be used as mouse-look. Boot up the game into any level. Look around. Notice that it 'appears' to lag ad you move the thumbs, but everything else about the game is running fine. This is NOT a performance issue.

Quit the game. Install TouchMousePointer, set it to fullscreen mode thru the taskbar icon. Boot up Portal again, notice how much smoother and accurate TMP emulates the mouse. This is a BIG problem for FPS and TPS games.

  • Slow updates:

I hope so! 3 months with only ONE very, very marginal update is very strange for users who game on Steam with updated content all the time. You guys will need to find a way to broaden your update roadmap to avoid looking like vapourware. It keeps people talking.

WOW, that was alot to type on a tablet touchscreen! I hope that expands alot of what was in the OP. Keep us updated!
