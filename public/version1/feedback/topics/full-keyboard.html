ArcaneAvis
10 months ago
  - In reply to HungryZealot
 

They keyboard is already able to be resized in game by using your fingers to scale in or out. :) It can also be repositioned.

Keystrokes are already tied to different actions in games, and each key may already be taken in-game, depending on the game itself. We can't have a button tied to the keyboard to bring up the virtual keyboard. The Home button was created to combine multiple functions that are needed in game into one button, thus freeing up screen space and the space the user has to customize their HUD.

I can definitely see how in a game like Scribblenauts, where a keyboard is needed for regular gameplay, that it would be efficient to have it mapped to a key. We felt that a keyboard shouldn't be needed frequently enough in most games that it would be all that difficult to use the Home button pull it up.

By design, the Virtual Controller doesn't function when the virtual keyboard is open. The problem with allowing the keyboard to function at the same time as the buttons is that if the keyboard is on top of buttons, we would have to determine which hit test takes presidence. Would you hit the keyboard key or would you hit the button behind it? Or would you hit them both at the same time? There are a lot of issues there.
