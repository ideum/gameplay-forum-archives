Something I just thought... many games I have already have means of being controlled entirely with a gamepad, usually, one that is XInput compatible (Microsoft Xbox 360 controller, Logitech F710, F310, etc...)

And I noticed that there are various "wrappers" available on the web to convert older DirectInput gamepad into XInput ones, or convert a PS3 gamepad into one that send XInput instructions... It seems that there are even various keyboard and mouse wrappers to convert keys and mouse movements into XInput instructions for games....

So, wouldn't it be awesome if we could use the GWG virtual controls to send XInput instructions to our games? Usually, games that support gamepads have all their menus already gamepad friendly too, so it would make it very easy to have profiles working for those type of games. In fact, we could even have a generic profile with all the buttons and sticks of a standard 360 gamepad that would provide a quick and easy way to play any game that supports a gamepad. Sure, that generic profile might get cluttered a bit with all the virtual buttons and sticks on screen, but it would work. Games that don't need all the buttons of a gamepad could then benefit from a more streamlined profile with better organized buttons. But still, the potential for this feature would be HUGE in my opinion.

For instance (some will call me crazy), the game XCOM: Enemy Unknown plays very very well with a gamepad to the point that I prefer playing with a gamepad than playing with kb+mouse. So having virtual controls that would make the game work exactly like it does when played with a gamepad would be a lot better in my opinion.

Is this something that could be done? Personnaly, I think this would make GWG exponentially more awesome than it already is.

> But...isn't GWG developed with XNA? > XNA uses XInput, not DirectInput.

The way it works now, is that virtual buttons can only map keyboard presses and mouse mouvement (and some other things like gestures if I'm not mistaken).

What I'm talking about, is be able to map XInput controls (Xbox360 gamepad inputs) to virtual on-screen buttons.

The benefit we would get out of this, is that many games have adapted control schemes for use with gamepads to navigate menus and other aspects of a game, instead of relying on a mouse cursor and clicks to navigate the same menus. I think this would make some games way easier to play with touch controls, if the underlying mode of communication with the game was gamepad emulation instead of keyboard + mouse emulation. Not for all games, but some games would truly benefit from this.

For example, I much prefer playing XCOM with a gamepad than with kb + mouse. I may be in the minority here, but the controls were fantastically adapted for the Xbox 360 gamepad and much more intuitive to me.

So having the option to map gamepad buttons and sticks to virtual on-screen controls would make GWG a lot better in my opinion. It would also help with games that differentiates left and right shift keys (like Pinball FX2) by using gamepad control for the GWG game profile instead of having to change default kb configuration in the game for GWG to work with it.

And regarding XNA, didn't Microsoft said that Windows 8 wouldn't support XNA? Or maybe they were talking about apps designed for the "modern" UI, as they also have to run on ARM devices...
