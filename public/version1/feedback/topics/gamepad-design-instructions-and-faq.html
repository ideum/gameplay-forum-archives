Hi willoemler,

We plan on making a video soon that gives some simplified instructions on making an optimal Virtual Controller for your game, plus some factoids on how gestures and hudgets work.

To start with, you need to know what all the keys for the particular game do so that you can map them in-tool. You can either find those online, within the game, or by messing around for a little while with a keyboard and mouse, noting what each default key command or mouse click does. You should have a pen and paper or just a word doc open for this. It's really useful to have those notes if you decide to upload your controller and want to write down what all your hudgets and gestures are mapped to in your game description.

You can get some ideas by looking at the layouts of our admin-submitted VCs.

As for gestures, you're right -- there is a large variety of combinations, and it can get a little confusing. Here is my attempt at explaining them:

  1. Tap - allows user to use 1-10 fingers* simultaneously on the screen to perform a key/mouse action (I usually use this for left/right mouse click or to fire a weapon).
  2. Hold-tap ** - use 1-10 fingers (either on hold or tap) to perform a key/mouse action (maybe good for a combo).
  3. Drag - allows user to use 1-10 fingers simultaneously to perform an action (usually for dragging the mouse "mouse hover" or move the camera, "mouse look" or moving the actual character).
  4. Rotate - use a rotate gesture with 2-10 different finger combinations to perform a key/mouse action (generally one that involves rotating the canvas, object, map, etc.).
  5. Scale - use 2-10 finger combinations to use a scale in/out gesture mapped to a key/mouse action (generally one that involves zooming in and out).

*Any gesture that involves more than 5-6 fingers to perform may cause the game to crash, so I generally don't recommend it.

**This gesture is a bit difficult for me personally

Hey willoemler!

We have a good amount of information on our Wiki page right now that you might find useful: http://wiki.gestureworks.com/index.php/GestureWorksGameplay:GestureWorks_Gameplay

Our software isn't emulating anything, but rather taking the controls already used in a game and applying them to on-screen touch buttons or gestures. Therefore, if the game itself does not have a point-and-click to "walk here" option, then we can't create that. We can't alter the way a video game plays.

However, if a game (such as Diablo) does have that option, then it's a simple matter of assigning that movement to either a button or a gesture. For instance, if the game you are playing has the "walk here" control set to left mouse button and you wanted to assign that to the tap Global Gesture, then you would simply create a tap Gesture and choose the left mouse button as its input. That way, whenever you tap on the screen, it's as if you were using the left mouse button at that location on the screen, which would make your character walk there.

The Modifiers function the same way for Gestures as they do on buttons. If you have a Modifier + an Input, it will act as if you are pressing those two things in game while actually only using one button or Gesture. As an example, the game DuckTales Remastered has an ability in game where you jump up on a Pogo stick. This is done by pressing Z and Spacebar simultaneously. So, using Gameplay, you could create a button where the Modifier could be the Z key and the Input could be Spacebar, so whenever you press that button in game, it will let you use the Pogo stick. Alternatively, you could create a tap Global Gesture with the same Modifier and Input, and then whenever you tap in that game, you'd jump on the Pogo stick.

Hope this helps explain things a little more! :)
