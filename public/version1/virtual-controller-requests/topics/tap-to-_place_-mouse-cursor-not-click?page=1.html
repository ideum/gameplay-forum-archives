yannickd
10 months ago
 

One recurring issue I see is that there are some games that need the mouse to hover somewhere without actually clicking. For example when casting some area spells in NeverWinter Nights 2 (a very good candidate for tablet games with the HD graphics), you need to hover the mouse to see where the spell would fall, and only then actually click.

So it would be nice in the gestures for example to have single finger tap: place mouse cursor, two-finger tap: left click. i.e. a new function "Place mouse cursor".

Of course, the joystick can usually be used to place the mouse, yet in some games results in the pointer being extremely ghostly/always disappearing such as DragonAge.

So low-hanging fruit there to fix issues that otherwise are a pain to look into (like why using the joystick doesn't draw the pointer correctly in some games). In the product I manage in my firm, I'd call that a papercut (of Ubuntu papercuts project fame), and have some time assigned to those low-hanging fruits each sprint/dev cycle.

Cheers,

Yannick

QuakeMatt

I've actually been thinking about something like this. Specifically, in the context of controlling XCOM: EU, but I guess it'd apply to many other RTS/TBS games.

My idea was that a single-finger touch could be used for basic mouse cursor positioning but, while pressed, would also become a control point for two further gestures - a second touch to the left of the control point would be translated into a left-click, and a touch to the right into a right-click.

While playing, you'd basically keep your middle finger on the screen all/most of the time to position the cursor, then tap (or drag) with your index and 3rd fingers to click with the left and right mouse buttons respectively.

Unfortunately, I can only guess as to how well this would work in practice! The gestures feel okay just poking away at my Surface, though.

It sounds like this may be a DPI scaling issue. To check and see if this would help with your cursor problem, right click on Trine.exe and go to Properties -> Compatibility tab -> and check the "Disable display scaling on high DPI settings" box.

We've done a lot of testing on Trine across multiple devices here and never came across that cursor issue, although the cursor in Trine can be moved manually by using the joystick. We definitely have been discussing including a hover option though and will talk more about this feature.

Thanks for the feedback!
