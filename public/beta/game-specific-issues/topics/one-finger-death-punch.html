I'm trying out a few of the games I had problems with on v1, so I'm just gonna create a unique topic for each game that still exhibits problems in the beta if that's ok... to help keep discussions/possible solutions separate.

My Test Device: Asus T100 32GB

I have a virtual controller I set up for One Finger Death Punch in GWG v1, which consists of...

  • A big button for each of left/right mouse buttons (one on each side of the screen)

  • A 'Tap' Global Gesture mapped to left-click, so you can navigate the level map

  • I was also experimenting with a virtual mouse-wheel to zoom in/out on the map, but not sure this is working quite right at the moment (may be may fault, so will ignore this for now!)

In GWG v1 this was how things turned out...

  • The game always launches in windowed mode (this is just how the game is, apparently no way to change this)

  • I can toggle to fullscreen mode from the initial menu in the game

  • The menus are navigable, although there is a problem which seems common to some games, where the 'Tap' global gesture is not performing the 'click' every time (instead it just moves the mouse cursor to that spot). It maybe only works 10-20% of the time, so mostly you have to double-tap to actually click menu options & select levels to play. This is obviously annoying, and can sometimes lead to you pressing twice on something when it already clicked the first time... which can trigger other unexpected things. This issue occurs whether 'Mouse Queue' is enabled or disabled.

  • Once you start a level, the game is perfectly playable via the on-screen hudgets/buttons... they react very well to taps, and never miss a tap/click like the global gesture does. Although sometimes when the levels speed up, it seems that the taps/clicks can't quite be processed/passed-through fast enough... like there's a little lag. I noticed that v2 includes a numeric setting to tweak the 'Key Queue' frequency, but there isn't a similar setting for the mouse? Maybe in the case of this specific game there are keys I can map to instead of mouse clicks for attacking left/right, but in general this could be an issue if games require many fast clicks in succession. It's particularly noticeable in this game when you fight the guys with crowns that popup the vertical stacks of fast clicks needed to kill them... sometimes they hit you because you can't click fast enough to hold them off. It doesn't seem to be a performance issue, as the framerate stays plenty smooth throughout.

With GWG BETA v1.11.1.0...

  • When I try to toggle the game to fullscreen at launch, I get a black screen / hang. I left it running for a while & nothing happened, so had to press 'Windows/Start' button & switch to Desktop to force-close.

  • If I leave it in windowed mode when launching, the game works fine, as before

  • Unfortunately, the same problem still exists with the Tap mapped to left-click Global Gesture (only occasionally clicking on the first tap).

  • And the game is equally playable, although the same 'slightly laggy' clicks are noticeable at faster speeds.

So the main 3 issues (with this game, at least)...

  • Hangs/crashes with the latest beta when entering fullscreen (didn't in GWG v1.0.7.1)

  • Tap gesture mapped to left-click almost always just moves mouse cursor to that point, but occasionally does perform the click too as it should.

  • On-Screen buttons mapped to left/right-click are responsive until you try to tap them rapidly, then they lag a little (they don't seem to 'lose' any taps, just fall a little behind in processing them through as mouse clicks)

I've seen the first issue (unreliable tap-click gesture) in a few other games, including 'Skulls of the Shogun'. But I'm fairly certain it's not an issue with my device/setup as I've successfully got the same gesture working reliably with every tap/click elsewhere, eg. to navigate menus & move your character in 'Torchlight'. And I've even tried tweaking Windows' Pen & Touch settings to no avail (disabled hold-to-right-click, reduced time delay for double-tap, etc). As I may have mentioned before, I have a 3rd-party util called 'TouchMousePointer' installed (you've probably heard of it), but completely removing that makes no difference.

I get the feeling I may be better off simply creating buttons for everything to map to X360 controls, but it seems a waste not to be able to navigate the level map by simply tapping your selection on the screen. And it would obviously be better to try to solve the cause of the problem, since it also affects other games.

Any suggestions/ideas?

Let me know if you need any more details. I didn't think to grab the logs at the time, but I'll try to remember to have a look when I next have a go, in case there are any clues regarding the fullscreen crash.

So I think GWG is already halfway there. I reckon it might have something to do with different ways mouse clicks can be detected, mouse down/up/both, or side effects of different API's being used.

EDIT: After thinking about it some more, here's what seems to be happening that would explain the double-tap thing.... each tap does a 'mouse up' at the previous location, then a 'mouse down' at the current location. What should happen is a 'mouse down' followed by immediate 'mouse up', both at the current tap location. Not sure on the other issue with the really sensitive zone in the middle of the screen though!
