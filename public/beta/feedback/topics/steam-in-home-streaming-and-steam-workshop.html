Hello everyone! I've already written about the capabilities of GWG working with Steam In-Home Streaming on the old forum, so I'm just gonna link the post: https://gameplay.gestureworks.com/gameplay_forums/feedback/topics/steam-in-home-streaming-compatibility And it still works in the beta. We just need some way of managing which controller should be launched with the streamed game.

I would also like to know if it's possible to utilize Steam Workshop for the controllers since it would drastically ease the subscribing and actually updating the controllers, give us the ability to comment and together improve the existing controllers, and you wouldn't have to approve all of them manually.

Edit: Also, you could try to add the beta as a branch under some kind of an access code which you may give to the beta testers afterwards. Then it's easy to opt into the beta by going to Steam Library, RMB on GWG -> Properties -> Betas and entering the access code. It would really ease the updating for us, beta folks.

Thanks for the feedback, these are excellent ideas! We're definitely going to be looking into taking advantage of more Steam features and the first one on the list is the Workshop for Virtual Controllers. We'll also take another look at loading up the correct controller when using In-Home Streaming.

We'd love to move beta to Steam, it would really help smooth the transition for testers once we go live and help us work out the kinks with Steam integration. We will try to make this happen during beta.

VC templates are a great idea and something we've talked about internally. This will definitely be on the short list for beta.

Thanks again for the suggestions!
