Hello guys, and thank you for inviting me for the beta! I hope I'll have enough time in the coming weeks to provide some useful feedback and maybe some controllers. My "test rig" will be a Dell venue 8 Pro 32GB version, with Steam and all of it's apps installed on a 64GB Sandisk Ultra microSD. Let me stop right here and point out, that during the setup process I could not install Gameplay on the SD-card. I was however able to move it and symlink back to the original install folder. This created exactly zero problems, so I guess the directory was preset in fear of installing it on a slow Class 4 card. I might test it later with one, just to see if it affects anything at all.

So far I've tested Dust:AET with a mobile as a controller and Teslagrad with a controller I previously created in Gameplay V1.

Both games worked well, Dust even worked without the previously needed hack to kill Steam before launch. Installing and pairing the phone to the tablet was straightforward (the phone is a Xiaomi Mi2 32GB running MIUI 4.5.23). Once paired the latency was neglectable, although while fighting it was hard to use the phone's controls, it takes some getting used to, since there is absolutely no feedback. I guess some vibration on touch would make it a bit easier and I'll definitely make some adjustments to the onscreen buttons. I might make a paper cutout as well, so that I can keep my fingers somewhere while I'm not pressing something, because this was more of a button mashing session rather than playing a game with any thought :)

One bug I noticed during (or rather after) playing with Dust, that after exiting the game it simply restarted. I mean I played for about 10 minutes (killed Gaius), then exited and in a few seconds Dust simply loaded again all by itself. After exiting a second time it didn't load again. I noticed it with Teslagrad as well, so this might need some looking into.

The phone client is easy to use, and pretty straightforward, but it's huge. 156MByte is simply too big, not many phones have 32GB internal memory like mine, so I'd slim it down before release, even 80MB would be pushing it. Please look into this! Other than that it was good, the power consumption was medium, since the screen was on, it did draw some power, I'd consider a 'blind mode' since usually I won't look at the phone at all, and it would save some power. I realize it would need to be a basic controller setup, where you can use a blind mode, maybe a simple two stick on the lower corners + four buttons on the top configuration, that would work with a lot of game out-of-the-box.

Carrying over my old controllers worked just fine, nothing to add here.

Welp, that's it after the firs hour, I'll try to get back as soon as I have more time!

Good work and thank you guys!

Thanks for the great feedback! I'll try to respond to each of your points.

Installation should be a little more open-ended once we move to Steam for distribution so hopefully this won't be an issue for too long.

On the remote app, you can open the "Options" menu and turn "Play Rumbles" ON. This will give you some haptic feedback when you hit a button. A paper cutout is a great idea!

The game restarting after you quit is definitely a bug that we'll look into.

You're right, the remote app is quite large. We're looking at slimming it and the desktop package both down in future releases. "Blind mode" is another good idea that we'll put on the board.

Thanks again for the feedback, keep it coming!
