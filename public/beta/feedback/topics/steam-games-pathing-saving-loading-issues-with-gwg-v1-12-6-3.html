I've noticed this issue with certain games on Steam with the latest beta version of GWG (although it may well affect other games, or all of them!)...

It appears that games are failing to find their save game / data file locations.... and are either crashing, or saving their data in the wrong place. See below for more details.

At this stage I suspect it's more likely this is an issue with Steam itself rather than GWG... perhaps it doesn't like being launched too early/during login (possibly before %appdata% has been properly mapped or something). Not entirely sure though. But I can't prevent Steam launching with GWG at login due to a bug in the GWG checkbox.

For information:-

  • Test Device = Asus T100

  • I have Steam installed onto SD card (as drive D:)

  • Steam games are all installed into the default library in that Steam installation location

I didn't have the problem until this version, but I previously set GWG to NOT auto-launch Steam at startup. That option won't disable for me any more... it keeps re-checking itself, so now it keeps starting Steam immediately when I reboot/login to Windows

If I close Steam, then restart Steam, the problem goes away, but... presumably GWG won't be hooked in properly in that case, unless I re-launch the GWG client and launch a Steam game from there so it restarts Steam? That's a lot of messing around whenever I have to reboot the machine.

The main thing I'd like to see is a fix to the 'auto-start at launch' option to properly allow me to disable it as I used to do. That should resolve it in my case.

See below for more info on the issue as I've seen it, in case anyone else reports similar problems. In all cases, closing, then reopening Steam fixes things...

  • I first noticed the problem with Little Inferno, which runs ok but wasn't finding my previous saves. Note that I'm not using a VC/overlay with this game any more, since GWG doesn't currently work with the game (I only needed to add an ESCAPE button to quit the game, but keys don't show up, the GWG semi-circle doesn't respond, and taps don't pass-through). So I'm just playing the game normally after GWG has launched Steam at startup. I eventually noticed that the game had created a new 'Little Inferno' data folder in the root of the SD card (D: drive). After trying various things, the only solution to this was to close & restart Steam. It then located the actual profiles that were stored in %AppData%. A complete reboot then made the issue resurface, and I can repeat this cycle every time.

  • I then looked for other games that save data to %AppData%

  • Torchlight crashes at launch with an error. My guess is the weird characters should really be a path to %AppData%, but something is corrupting them...

    "Settings were unable to save. Attempted to save settings at: [WEIRD SEQUENCE OF CHARACTERS HERE!]/Runic Games/Torchlight/"

    Windows Error: No such file or directory.

  • Unity of Command crashes at launch with an error. Note that I'm not actually using any GWG VC/overlay. Again, looks like some corruption in the path... not sure if the double-backslashes are an intentional/accidental string-escaping thing, or part of the corruption, but the little 'u' could be a symptom too...?

    The logfile 'C:\Windows\system32\config\systemprofile\Desktop\Unity of Command CRASH.log' could not be opened:

    [Errno 2] No such file or directory:

    u'C:\\Windows\\system32\\config\\systemprofile\\Desktop\\Unity of Command CRASH.log'
