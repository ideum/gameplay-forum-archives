I've just submitted a bug report. I've also got some additional feedback for the GWG config UI:

  • Sliders like scale, sensitivity, opacity and the like are now bigger, but the up/down arrows are unusable on a touch device. Please provide a more touch-friendly approach for this
  • Please add more items to the Glyph selection, like directional arrows, Function Keys and such
  • Also please provide a Copy function to copy a button with its characteristics like opacity and scale
  • When selecting multiple buttons and changing opacity, only some will get the opacity you selected, others will not and yet others will get a random value it seems
  • Switches don't always work as expected. In Borderlands 2 for instance, I use a switch for crouch. Toggling on works fine, but off doesn't. I need to Jump to get out of crouch mode.
  • What would be cool is that when using the in-game edit mode, you would allow to re-assign keys and copy buttons as well, so that building a new controller doesn't require as much restarting of the game. Also, when returning from a game after having edited in-game, just reload the controller instead of notifying of in-game changes.

I will keep this flow running as I am testing my games/controllers. Keep up the good work guys, the responsiveness of GWG in game has improved a lot. In the previous version I couldn't play Borderlands 2 at all, now only the aiming needs some tweaking. Only thing I struggle a lot with is that for moving you need a pretty high sensitivity, but for aiming it might be a lot lower. And the gradual sensitivity of the D-Pad doesn't really help all that well (for me).
